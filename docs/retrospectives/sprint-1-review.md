## Stakeholder Feedback
- Positive feedback:

The login flow was smooth and consistent with existing internal tools, which stakeholders appreciated.

The filter-by-capacity feature received strong praise; stakeholders noted it would save significant time for event organisers.

The room list and availability calendar were described as "exactly what we envisioned" by the facilities manager.



- Concerns or questions:

There was concern about what happens when two users attempt to book the same room simultaneously; team acknowledged this is part of User Story#05 acceptance criteria.

One stakeholder questioned the absence of a cancellation feature — not yet in the backlog.

Stakeholders asked whether the booking system would integrate with Google Calendar or Outlook — this was not in the original scope and needs Product Owner clarification


- Requests or adjustments:

Add cancellation reason field in Sprint 2

Request to add a "favourite rooms" shortlist feature — logged as a new backlog item.

Request to display room amenities (projector, whiteboard, video conferencing) on the room list — User Story#1 acceptance criteria may need to be expanded or a new story created.

Stakeholders would like to see end-to-end booking demonstrated at the Sprint 2 review as the top priority.
